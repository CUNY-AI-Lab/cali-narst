## What This Is

A custom HTML slide deck used for CUNY AI Lab faculty workshops. All content lives in `index.html`. `SLIDES.md` is a markdown mirror kept in manual sync. No build system. No npm.

---

## Cardinal Rules

- **`index.html` is the source of truth.** All slide edits go there.
- **Keep `SLIDES.md` in sync** after every content change to `index.html`.
- **Never break accessibility.** WCAG 2.1 AA is a hard requirement, not a preference. See the checklist below.
- **No em dashes (`—`)** anywhere in slide content.
- **No Co-Authored-By lines** in commits.
- **Commit messages:** short, lowercase, no sign-off. Example: `update slide 12 with stronger knowledge example`

## How to Edit Slides

Slides are `.slide` divs in `#deck` inside `index.html`. Each slide must have all four attributes:

```html
<div class="slide layout-split" role="group" aria-roledescription="slide" aria-label="Slide N: Title" tabindex="-1">
```

**To add a slide:**
1. Copy the nearest slide with the same layout as your target
2. Give it the next sequential number in `aria-label`
3. Update the scrubber total in `deck-engine.js` if needed (search for `totalSlides`)
4. Add a corresponding section to `SLIDES.md`

**To edit a slide:**
1. Edit the `.slide` div in `index.html`
2. Mirror the change in `SLIDES.md`
3. Check that all `aria-label` values still accurately describe slide content

**To remove a slide:**
1. Delete the `.slide` div
2. Renumber all subsequent `aria-label="Slide N: ..."` attributes
3. Update `SLIDES.md`

## Progressive Reveal

Two mechanisms, both handled by `deck-engine.js`:

- **Step reveal:** Add `class="step-hidden" data-step="N"` to any element. It reveals in order on forward advance.
- **Stream bullets:** Wrap `<li>` items in `<ul class="stream-list">` for staggered entry animation.

## Carousel

To add a screenshot carousel in a `.stage` panel:

```html
<div class="carousel" data-interval="8000">
  <div class="carousel-item active">
    <img src="images/slideN-a.png" alt="Descriptive alt text">
    <div class="carousel-caption"><strong>Step Label</strong><br>One sentence describing the screenshot.</div>
  </div>
  <div class="carousel-dots"></div>
</div>
```

- First item must have `class="carousel-item active"`
- All images need meaningful alt text (not empty, not "screenshot")
- Dots and arrows are auto-generated by `carousel.js`

## Accessibility Checklist

Run this before every commit that touches `index.html`:

- [ ] Every `.slide` has `role="group" aria-roledescription="slide" aria-label="Slide N: Title" tabindex="-1"`
- [ ] No text uses opacity below `0.65` on dark backgrounds (`--light-text-50` = 0.65, not 0.5)
- [ ] Decorative emoji have `aria-hidden="true"`
- [ ] Decorative SVGs have `aria-hidden="true"` and `focusable="false"`
- [ ] Step-hidden elements use both `opacity: 0` and `visibility: hidden`
- [ ] New interactive elements are keyboard-navigable
- [ ] `prefers-reduced-motion` is respected in any new animation

## Design Tokens

All colors are in `css/styles.css` under `:root`. Do not hardcode hex values — use the tokens:

- Backgrounds: `--bg-light`, `--bg-panel`, `--dark-bg`, `--dark-bg-lighter`
- Text: `--text-dark`, `--text-muted`, `--light-text`, `--light-text-50`
- Accents: `--accent-navy`, `--accent-blue`, `--accent-cyan`, `--accent-light-blue`, `--accent-red`, `--accent-amber`, `--accent-green`

Typography: `Fraunces` (headings), `DM Sans` (body), `JetBrains Mono` (code/prompts) — loaded from Google Fonts CDN.

## Prompt Blocks

Prompt quality examples use these classes:

```html
<div class="prompt-block prompt-bad">
  <span class="prompt-label">Weak</span>
  <p>...</p>
</div>
```

Classes: `prompt-bad`, `prompt-mid`, `prompt-good`. Always include a `.prompt-label` badge.

## Cloning and Starting a New Deck

```bash
# Clone the reference repo
git clone https://github.com/CUNY-AI-Lab/knowledge-collections.git my-new-deck
cd my-new-deck

# Remove the upstream remote and point to a new repo
git remote set-url origin https://github.com/YOUR-ORG/your-new-repo.git

# Edit content
# - Update title, date, and roadmap in slides 1-2
# - Replace or reuse discipline examples in the body slides
# - Swap images in images/ as needed

# Sync SLIDES.md after edits
# Deploy: push to main, enable GitHub Pages at root

git add -A && git commit -m "initial custom deck" && git push -u origin main
```

## What Not to Do

- Don't add npm, webpack, or any build step
- Don't add framework dependencies (React, Vue, etc.)
- Don't strip or stub out accessibility attributes
- Don't use absolute file paths in image `src` (relative paths only)
- Don't commit unoptimized images over ~300KB without reason
- Don't edit `SLIDES.md` without editing `index.html` first (or at the same time)
