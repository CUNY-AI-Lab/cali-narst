# Frontend Design Skill

This skill governs all visual/aesthetic changes to the slide deck. Load it before touching any CSS or layout attribute.

## When to Use

- Editing `css/styles.css`, `css/responsive.css`, or `css/animations.css`
- Changing layout classes on `.slide` elements
- Adding or modifying colors, spacing, typography, or shadows
- Adding new visual components (cards, badges, callouts, flow diagrams)
- Changing font sizes, weights, or line heights
- Adjusting any `--accent-*` or other design token values

## Design System

### Tokens (always use these, never hardcode hex values)

**Backgrounds:**
- `--bg-light` — main content background (light)
- `--bg-panel` — stage/panel background (slightly darker)
- `--dark-bg` — dark slide background
- `--dark-bg-lighter` — slightly lighter dark, for layering
- `--code-bg` — inline code and prompt block background

**Text:**
- `--text-dark` — primary text on light backgrounds
- `--text-muted` — secondary/de-emphasized text
- `--light-text` — primary text on dark backgrounds
- `--light-text-50` — de-emphasized text on dark (minimum 0.65 opacity — do not reduce)

**Accents:**
- `--accent-navy` — primary brand color
- `--accent-blue` — interactive / highlight
- `--accent-cyan` — secondary highlight
- `--accent-light-blue` — subtle accent
- `--accent-red` — warning / weak examples
- `--accent-amber` — caution / mid-level examples
- `--accent-green` — success / strong examples

### Typography

| Role | Font |
|---|---|
| Headings | `Fraunces` (Google Fonts CDN) |
| Body | `DM Sans` (Google Fonts CDN) |
| Code / Prompts | `JetBrains Mono` (Google Fonts CDN) |

Do not change font families. Do not add new Google Fonts imports without explicit instruction.

### Slide Layouts

| Class | Structure | Use case |
|---|---|---|
| `layout-split` | `.content` (left) + `.stage` (right panel) | Most instructional slides |
| `layout-content` | Single column, light | Text-heavy, no visual aid needed |
| `layout-full-dark` | Centered, dark | Roadmaps, summaries |
| `layout-divider` | Large centered heading, dark | Section breaks |
| `layout-grid` | `.grid-2` cards, light | Comparisons, feature lists |

## Accessibility Requirements (Non-Negotiable)

Every visual change must preserve these:

- **Contrast:** Text on dark backgrounds must use `--light-text-50` at minimum 0.65 opacity. Never set text opacity below 0.65 on dark backgrounds.
- **Focus visible:** All interactive elements need `:focus-visible` styles. Do not remove or override existing focus ring styles.
- **Reduced motion:** Any new `@keyframes` animation must have a corresponding block in `css/animations.css` under `@media (prefers-reduced-motion: reduce)` that disables or stubs it.
- **Decorative content:** Emoji and decorative SVGs must have `aria-hidden="true"`. Decorative SVGs also need `focusable="false"`.
- **Step-hidden elements:** Must use both `opacity: 0` AND `visibility: hidden` so unrevealed content is hidden from screen readers.

## Component Patterns

### Prompt Blocks

```html
<div class="prompt-block prompt-good">
  <span class="prompt-label">Strong</span>
  <p>Prompt text here.</p>
</div>
```

Classes: `prompt-bad` (red), `prompt-mid` (amber), `prompt-good` (green). Always include `.prompt-label`.

### Tip / Callout Boxes

```html
<div class="tip-box">
  <strong>Tip:</strong> One sentence of advice.
</div>
```

### Cards (grid layout)

```html
<div class="grid-2">
  <div class="card">
    <h3>Card Title</h3>
    <p>Card body.</p>
  </div>
</div>
```

### Flow Diagrams

Use inline SVG for arrows. All SVGs must have `aria-hidden="true"` and `focusable="false"`. Keep shapes simple — chevron arrows, horizontal flows.

### Carousel

```html
<div class="carousel" data-interval="8000">
  <div class="carousel-item active">
    <img src="images/slideN-a.png" alt="Descriptive alt text">
    <div class="carousel-caption"><strong>Step Label</strong><br>One sentence describing the screenshot.</div>
  </div>
  <div class="carousel-dots"></div>
</div>
```

First item: `class="carousel-item active"`. Dots and arrows are auto-generated by `carousel.js`.

## What Not to Do

- Do not hardcode hex values — always use CSS tokens
- Do not add new font families without explicit instruction
- Do not remove or weaken existing `:focus-visible` styles
- Do not use `visibility: hidden` without also setting `opacity: 0` on step-hidden elements
- Do not set text opacity below 0.65 on dark backgrounds
- Do not add new animations without a `prefers-reduced-motion` override
- Do not use `em` dashes (`—`) in any slide text content
- Do not use inline `style=""` attributes except for dynamic JS-driven values
